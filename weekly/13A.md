**1. Introduction & Problem (1 min)**
- Introduce team members
- State app name and core problem you're solving
- Identify your target audience (reference personas)

**2. Research & Design Evolution (1-2 min)**
- Show the original app proposal/infographic that inspired you
- Brief walkthrough of design evolution:
  - 1-2 lo-fi screens ‚Üí corresponding hi-fi screens
  - How feedback from testing improved the design

**3. Live Demo (2-3 min)** ‚≠ê **MAIN FOCUS**
- Walk through 2-3 core user flows
- Show key interactive elements in action
- Explain navigation and main features
- Highlight your best design decisions

**4. Testing & Iteration (1 min)**
- Brief overview of testing process (lo-fi and 12B peer feedback)
- Share 2-3 specific improvements from user feedback:
  - "Users couldn't find X, so we added Y"
  - "Navigation was confusing, so we changed Z"

**5. Conclusion (30 sec)**
- Summarize project impact
- "Imagine if..." statement about the app's potential
- Thank the audience and invite questions
